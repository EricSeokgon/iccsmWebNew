① 표준보안API 설치파일 시스템에 올리기

 

운영관리시스템으로부터 다운받은 표준보안API 설치파일을 보안 서비스를 제공할 시스템에 올린다. FTP로 설치파일을 올릴때는 반드시 “binary” 모드로 올려야 한다.

 

② 설치파일 풀기

 

시스템에 올라간 설치파일의 압축을 푼다.

 

- Windows인 경우 : 압축 프로그램을 이용하여 푼다.

- Windows가 아닌경우

           >> gunzip libgpkiapi*.tar.gz                // 압축 풀기

           >> tar xvf libgpkiapi*.tar                    // tar 풀기

 

③ 표준보안API 설치 위치에 옮기기

 

표준보안API가 구동될 위치를 정하고 표준보안 API 모듈(ex. libgpkiapi.so, libgpkiapi_jni.so, libgpkiapi_jni.jar)과 환경파일 (gpkiapi.conf)을 해당 위치로 옮긴다. (만약, 설치파일에 포함되어있는 LDAP 라이브러리가 해당 시스템에 설치되어있지 않다면 LDAP 라이브러리도 함께 설치한다.)

 

④ 라이브러리 경로 잡기

 

표준보안API 모듈을 옮긴 위치가 라이브러리 경로에 잡혀있지 않으면 해당 위치의 라이브러리 경로와 클래스 경로를 잡아준다. (2.2 참조)

 

⑤ 테스트 프로그램 구동하기

 

설치파일에 포함되어있는 테스트 프로그램을 구동하여 정상적으로 실행됨을 확인한다.
